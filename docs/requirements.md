# 要件定義書 — Solana Bot Trading Terminal (Mock)

## 1. プロジェクト概要

Solana チェーン上のトークンを対象とした **Bot 自動売買トレーディングターミナル** のモック画面を作成する。
実際のブロックチェーン接続や取引は行わず、**見た目上リアルに自動売買が動作しているように見える UI** を構築する。

### 1.1 プロジェクト名

`NovaPlex Core — Automated Trading Terminal`

### 1.2 目的

- Bot による自動売買が稼働中であるかのようなリアルタイム風 UI を提示する
- 勝率 **60〜70%** のパフォーマンスを表示し、現実的な成果に見せる
- プロフェッショナルな暗号通貨トレーディングプラットフォームと遜色ない UI 品質を実現する

## 3. 非機能要件

### 3.1 パフォーマンス

- 60fps での滑らかなアニメーション
- 初回ロード 3 秒以内
- メモリリークなし (長時間表示を想定)

### 3.2 レスポンシブ

- 最低対応幅: 1280px (トレーディング UI のため PC メイン)
- 推奨: 1920x1080 フル HD

### 3.3 ブラウザ対応

- Chrome (最新), Firefox (最新), Edge (最新)

### 3.4 アクセシビリティ

- ダークテーマをデフォルト (トレーディング UI の慣例)
- 高コントラストな利益/損失の色分け

---

## 4. 数値的リアリティ仕様

### 4.1 勝率シミュレーション

- 全トレード中の **勝ちトレード比率: 60〜70%**
- 勝ちトレードの平均利益: **+2.5%〜+8.0%**
- 負けトレードの平均損失: **-1.5%〜-5.0%**
- リスクリワード比: **約 1:1.3〜1:1.8**

### 4.2 トレード頻度

- 1 時間あたり **3〜8 トレード** 生成
- ピーク時 (ボラティリティ高) はやや多め
- 深夜帯 (表示上) は少なめ

### 4.3 ポートフォリオ推移

- 初期残高: **50 SOL** (表示上)
- 日次リターン: **+0.5%〜+2.5%** (平均)
- 時折 **-0.3%〜-1.2%** のドローダウン日を混ぜてリアリティを確保
- 月間リターン: **+15%〜+40%** のレンジで推移
- 初期表示の総 PnL は設定画面で上書き可能 (現実的レンジに制限)

### 4.4 対象トークン

実在の Solana エコシステムトークンの **名前のみ** を使用。
初期表示はローカル候補を使い、起動後に PumpPortal の新規トークン名で差し替える:

| ティッカー | トークン名 | 用途 |
|-----------|-----------|------|
| SOL | Solana | ベース通貨 |
| RAY | Raydium | DEX トークン |
| JUP | Jupiter | アグリゲーター |
| BONK | Bonk | ミームコイン |
| WIF | dogwifhat | ミームコイン |
| JTO | Jito | リキッドステーキング |
| PYTH | Pyth Network | オラクル |
| ORCA | Orca | DEX |
| MNDE | Marinade | ステーキング |
| TENSOR | Tensor | NFT マーケット |

## 5. トークン名データソース仕様

### 5.1 優先ソース

- 1st: PumpPortal WebSocket `wss://pumpportal.fun/api/data`
- 購読メソッド: `subscribeNewToken`
- 利用用途: **新規トークン名 / シンボル / mint の取得のみ**

### 5.2 取得ルール

- 起動時に WebSocket 接続し、一定件数 (例: 20 件) を収集
- 重複 `mint` は除外
- `name` または `symbol` が欠損の場合は採用しない
- UI には上限件数 (例: 10 件) だけ表示

### 5.3 フォールバック

- 接続失敗・タイムアウト時はローカル静的トークン名リストを使用
- 再接続は指数バックオフで実施
- フォールバック中でも UI は通常動作を維持

---

## 6. 制約事項

- **取引データは外部 API に依存しない** — 価格・損益・約定はクライアントサイドモック生成
- **例外としてトークン名のみ PumpPortal を利用可能**
- **ウォレット接続機能は実装しない** — 表示のみ
- **実際のトランザクションは発行しない**
- ブロックチェーンの状態は一切参照しない
- 設定値は UI 表示/モック挙動にのみ反映し、実取引には関与しない

## 8. 用語定義

| 用語 | 定義 |
|------|------|
| PnL | Profit and Loss (損益) |
| Bot | 自動売買を行うプログラム (本プロジェクトではモック) |
| エントリー | ポジションを建てる (買い) |
| エグジット | ポジションを閉じる (売り) |
| ドローダウン | 最大資産からの下落率 |
| Sharpe Ratio | リスク調整後リターンの指標 |
| スリッページ | 注文時と約定時の価格差 |
| 初期表示 PnL | 画面表示開始時点の `Total PnL` 初期値 |
| アニメーション速度 | UI 演出の再生倍率 (`0.5x` 〜 `1.5x`) |
| テーマカラー | アクセントカラーセット (例: Emerald / Blue / Violet) |

---

## 9. 設定画面仕様 (追加)

### 9.1 設定項目

- **初期表示総 PnL (Total PnL Start Value)**
  - 型: 数値 (SOL)
  - 許容範囲: `-5.00` 〜 `+30.00` SOL
  - 初期値: `+12.38` SOL
  - 補助: `%` は初期残高から自動算出
- **アニメーション速度**
  - 選択肢: `0.75x` / `1.00x` / `1.25x`
  - 初期値: `1.00x`
- **テーマカラー**
  - 選択肢: `Emerald (既定)` / `Blue` / `Violet` / `Amber`
  - 適用対象: accent color とバッジ、リンク、ハイライト

### 9.2 反映ルール

- 設定変更は「即時プレビュー」+「保存」で確定
- `初期表示総 PnL` は「次回シミュレーション開始時」に適用
- `アニメーション速度` と `テーマカラー` は即時反映
